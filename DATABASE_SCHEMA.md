# Database Spec and Schema

This document describes `data/usage_stats.sqlite`, the SQLite database generated by
`src/pipeline.py` from the monthly Excel workbook in `raw/`.

The database file is generated output and is ignored by git. The durable schema
contract is the `initialize_database()` function in `src/pipeline.py`.

## Logical Model

The database is intentionally simple:

| Object | Type | Purpose |
| --- | --- | --- |
| `observations` | Table | Fact table. One row per metric, dataset identity, month, channel, and service combination. |
| `import_runs` | Table | Audit table for successful workbook imports. |
| `observations_chronological` | View | Convenience view over `observations`, ordered by year, month, and insertion index. |
| `sqlite_sequence` | SQLite internal table | Tracks autoincrement counters. Not part of the application schema. |

There are no foreign keys and no triggers.

## Ingestion Contract

The loader expects one `.xlsx` workbook, unless an explicit input path is passed.
The workbook must contain these worksheets:

| Worksheet | Stored `metric` |
| --- | --- |
| `API Calls` | `api_calls` |
| `Downloads` | `downloads` |

The first three workbook headers must be:

| Source header | Stored column |
| --- | --- |
| `Data Provider` | `data_provider` |
| `Dataset Name` | `dataset_name` |
| `Data Nature` | `type` |

The loader discovers these metadata columns by heading:

| Source header | Stored column |
| --- | --- |
| `Access Group` | `access_group` |
| `Remarks` | `remarks` |
| `ASDPID` | `asdp_id` |

Month columns are discovered from headings such as `May 2026` or `Jun 2026`.
Trailing note markers such as `*1` are ignored when parsing month headings.

Rows are loaded when they have a data provider and dataset name, unless the data
provider is an annotation marker such as `#General Remarks:` or `*1`, or the
data type normalizes to `total`. A blank `Data Nature` does not automatically
make a row invalid; the loader keeps such rows when the row still has usage
values or dataset metadata such as `Access Group`, `Remarks`, or `ASDPID`.

## Dimension Rules

Each source measure cell becomes one `observations` row.

| Dimension | Rule |
| --- | --- |
| `year`, `month` | Parsed from the current month heading. |
| `metric` | Derived from the worksheet name. |
| `channel` | Taken from header rows when the header is `B/Ds (GNET)` or `Public (Internet)`. Can be `NULL`. |
| `service` | For `API Calls`, taken from nearby header text such as `Data Query Service` or `GeoSpatial Services`; can be `NULL`. For `Downloads`, stored as `NULL`. |
| `source_row`, `source_column` | 1-based workbook coordinates for the source cell. |

## Value Rules

Source cells are normalized into `value`, `value_status`, and `raw_value`.

| Source cell | `value` | `value_status` | `raw_value` |
| --- | --- | --- | --- |
| Blank or empty string | `NULL` | `missing` | `NULL` |
| Boolean | `0` or `1` | `reported` | Original boolean text |
| Integer or float | Numeric value | `reported` | Numeric text |
| Numeric text, with commas allowed | Parsed numeric value | `reported` | Original text |
| `N/A`, `NA`, `NOT APPLICABLE` | `NULL` | `not_applicable` | Original text |
| Other text | `NULL` | `other` | Original text |

`value` uses SQLite `NUMERIC` affinity. Whole-number floats are stored as
integers by the loader.

## Identity and Upsert Rules

`dataset_identity` is derived before the record key is computed:

| Source condition | `dataset_identity` format |
| --- | --- |
| `asdp_id` is present | `asdp:<normalized_asdp_id>` |
| `asdp_id` is missing | `fallback:<normalized_data_provider>|<normalized_dataset_name>|<normalized_type>` |

Normalization lowercases text, trims outer whitespace, and collapses internal
whitespace.

`record_key` is a SHA-256 hex digest of a compact JSON array containing:

```text
[
  metric,
  dataset_identity,
  year,
  month,
  channel or "",
  service or ""
]
```

`record_key` is unique and is the database-level idempotency key. Re-importing
the same workbook does not create duplicates. If an existing `record_key` has
changed semantic fields, the row is updated and `updated_at` is refreshed.

Semantic fields used to detect changes:

```text
data_provider, dataset_name, year, month, channel, type, service,
remarks, asdp_id, metric, value, value_status, raw_value, access_group,
dataset_identity
```

Rows that disappear from a later workbook are not deleted.

## Table: observations

Primary fact table.

| Column | Type | Required | Default | Notes |
| --- | --- | --- | --- | --- |
| `index` | `INTEGER` | No | None | Autoincrement primary key. Quote as `"index"` in SQL when needed. |
| `data_provider` | `TEXT` | Yes | None | Source `Data Provider`. |
| `dataset_name` | `TEXT` | Yes | None | Source `Dataset Name`. |
| `year` | `INTEGER` | Yes | None | Check constraint: `2000` to `2100`. |
| `month` | `INTEGER` | Yes | None | Check constraint: `1` to `12`. |
| `channel` | `TEXT` | No | None | Usually `B/Ds (GNET)` or `Public (Internet)`. |
| `type` | `TEXT` | No | None | Source `Data Nature`. |
| `service` | `TEXT` | No | None | API service dimension when available. |
| `remarks` | `TEXT` | No | None | Source `Remarks`. |
| `asdp_id` | `TEXT` | No | None | Source `ASDPID`. |
| `metric` | `TEXT` | Yes | None | Check constraint: `api_calls` or `downloads`. |
| `value` | `NUMERIC` | No | None | Parsed numeric usage value, or `NULL` for non-reported statuses. |
| `value_status` | `TEXT` | Yes | None | Check constraint: `reported`, `missing`, `not_applicable`, or `other`. |
| `raw_value` | `TEXT` | No | None | Original value text for reported text, not-applicable markers, and other text. |
| `access_group` | `TEXT` | No | None | Source `Access Group`. |
| `source_file` | `TEXT` | Yes | None | Workbook filename at import time. |
| `source_sheet` | `TEXT` | Yes | None | Worksheet name. |
| `source_row` | `INTEGER` | Yes | None | 1-based worksheet row. |
| `source_column` | `INTEGER` | Yes | None | 1-based worksheet column. |
| `dataset_identity` | `TEXT` | Yes | None | Stable dataset identity used by the loader. |
| `record_key` | `TEXT` | Yes | None | Unique SHA-256 idempotency key. |
| `updated_at` | `TEXT` | Yes | `CURRENT_TIMESTAMP` | SQLite timestamp of insert or semantic update. Stored without a timezone offset. |

Canonical DDL:

```sql
CREATE TABLE observations (
  "index" INTEGER PRIMARY KEY AUTOINCREMENT,
  data_provider TEXT NOT NULL,
  dataset_name TEXT NOT NULL,
  year INTEGER NOT NULL CHECK (year BETWEEN 2000 AND 2100),
  month INTEGER NOT NULL CHECK (month BETWEEN 1 AND 12),
  channel TEXT,
  type TEXT,
  service TEXT,
  remarks TEXT,
  asdp_id TEXT,
  metric TEXT NOT NULL CHECK (metric IN ('api_calls', 'downloads')),
  value NUMERIC,
  value_status TEXT NOT NULL CHECK (value_status IN ('reported', 'missing', 'not_applicable', 'other')),
  raw_value TEXT,
  access_group TEXT,
  source_file TEXT NOT NULL,
  source_sheet TEXT NOT NULL,
  source_row INTEGER NOT NULL,
  source_column INTEGER NOT NULL,
  dataset_identity TEXT NOT NULL,
  record_key TEXT NOT NULL UNIQUE,
  updated_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

## Table: import_runs

Audit table. One row is inserted after each successful import transaction.

| Column | Type | Required | Default | Notes |
| --- | --- | --- | --- | --- |
| `run_id` | `INTEGER` | No | None | Autoincrement primary key. |
| `input_file` | `TEXT` | Yes | None | Workbook filename. |
| `workbook_sha256` | `TEXT` | Yes | None | SHA-256 digest of the workbook bytes. |
| `started_at` | `TEXT` | Yes | None | Import start timestamp from Python, ISO 8601 with UTC offset. |
| `completed_at` | `TEXT` | Yes | None | Import completion timestamp from Python, ISO 8601 with UTC offset. |
| `rows_seen` | `INTEGER` | Yes | None | Number of candidate observation records generated from the workbook. |
| `inserted_rows` | `INTEGER` | Yes | None | Number of new observation rows inserted. |
| `updated_rows` | `INTEGER` | Yes | None | Number of existing observation rows changed. |
| `unchanged_rows` | `INTEGER` | Yes | None | Number of existing observation rows skipped as unchanged. |
| `min_date` | `INTEGER` | Yes | None | Earliest imported month in `YYYYMM` form. |
| `max_date` | `INTEGER` | Yes | None | Latest imported month in `YYYYMM` form. |

Canonical DDL:

```sql
CREATE TABLE import_runs (
  run_id INTEGER PRIMARY KEY AUTOINCREMENT,
  input_file TEXT NOT NULL,
  workbook_sha256 TEXT NOT NULL,
  started_at TEXT NOT NULL,
  completed_at TEXT NOT NULL,
  rows_seen INTEGER NOT NULL,
  inserted_rows INTEGER NOT NULL,
  updated_rows INTEGER NOT NULL,
  unchanged_rows INTEGER NOT NULL,
  min_date INTEGER NOT NULL,
  max_date INTEGER NOT NULL
);
```

## View: observations_chronological

Convenience view for stable chronological reads.

```sql
CREATE VIEW observations_chronological AS
  SELECT * FROM observations ORDER BY year, month, "index";
```

Use this view when row order matters:

```sql
SELECT *
FROM observations_chronological;
```

## Indexes

| Index | Columns | Unique | Purpose |
| --- | --- | --- | --- |
| `sqlite_autoindex_observations_1` | `record_key` | Yes | Backs the unique constraint on `record_key`. |
| `observations_date_idx` | `year`, `month`, `index` | No | Chronological filtering and ordering. |
| `observations_metric_date_idx` | `metric`, `year`, `month` | No | Metric-specific monthly queries. |
| `observations_asdp_idx` | `asdp_id` | No | Dataset lookup by ASDPID. |
| `observations_provider_idx` | `data_provider` | No | Provider filtering. |

## Common Queries

Latest successful import:

```sql
SELECT *
FROM import_runs
ORDER BY run_id DESC
LIMIT 1;
```

Monthly totals by metric:

```sql
SELECT
  year,
  month,
  metric,
  SUM(value) AS total_value
FROM observations
WHERE value_status = 'reported'
GROUP BY year, month, metric
ORDER BY year, month, metric;
```

Reported usage for one ASDPID:

```sql
SELECT
  year,
  month,
  metric,
  channel,
  service,
  value
FROM observations_chronological
WHERE asdp_id = :asdp_id
  AND value_status = 'reported';
```

Rows requiring follow-up:

```sql
SELECT
  metric,
  raw_value,
  COUNT(*) AS rows
FROM observations
WHERE value_status = 'other'
GROUP BY metric, raw_value
ORDER BY rows DESC;
```

## Maintenance Notes

Schema creation is idempotent because it uses `CREATE TABLE IF NOT EXISTS` and
`CREATE INDEX IF NOT EXISTS`. The chronological view is dropped and recreated
on each normal loader run.

View columns inherit from `observations`, but view constraints and defaults are
not enforced by the view itself.

`import_runs.started_at` and `import_runs.completed_at` are Python ISO 8601 UTC
timestamps with a `+00:00` offset. `observations.updated_at` is a SQLite
`CURRENT_TIMESTAMP` text value without a timezone offset.

The project currently has no migration framework. If a table definition changes,
existing database files may need to be purged and rebuilt from the workbook, or
migrated manually.
